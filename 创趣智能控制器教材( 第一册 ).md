## 第1节 人与计算机间的翻译员 - 编程软件

世界上第一台计算机于1946年发明，计算机从诞生至今，随着科学技术的飞速发展，早已成为了我们生活中必不可少的一部分。很多时候，我们都需要和计算机“打交道”，人类相互之间的交流可以通过各种语言，比如汉语，英语和法语等，`配图`我们把这些人类用于交流的语言也称为自然语言，而要想让计算机明白我们在“说什么”， 甚至能够按照我们的想法去“做什么”，就要运用到计算机语言，计算机语言可以理解是遵循一些规则的贴近于自然语言，同时又与计算机运行存在联系的语言，将用计算机语言编辑好的脚本（也叫程序）再通过编程软件的转化，就能得到真正可以操控计算机的指令。`配图`

如何编写一段脚本（程序）呢？

在创趣编程软件中，将构建脚本需要的单个模块从模块区，拖拽到脚本区，对应模块的组合，即完成了编写的过程。当然，编写也同样需要讲究一些规则，这个就好比我们传统的代码编程中的语法，但这里的“语法”大多数都不需要我们去特别记忆，也是一种图形化的展现形式。我们在拼搭积木时，并不是所有的积木都能连接在一起，积木会存在不同的形状不同的连接位置。软件中的模块亦是如此，一个符合“语法”规则的程序，会在脚本区呈现一种金米相连的状态`图示`，并且当其中一个模块靠近另一个模块时，你会发现像是有磁力作用一样，被吸附在一起；而不符合“语法”规则的程序，是无法被衔接在一起的`图示`

## 第1节 数据的家 - 变量

无论是在屏幕上显示一段话，还是完成一个复杂的计算，在程序中用到的“话”或者“数字”都是数据的一种表现形式。实际上， 编写程序的过程，就是寻找数据间关系，并用计算机语言表示出的过程。对于程序而言，有些数据可能很庞大且需要反复使用，有些数据可能不确定要依赖于别的数据，并不是所有的数据都会像“你好世界”这样可以很容易地直接表达。那么，有什么更好的办法记录和运用它们呢？

`图示`原来，计算机的大脑就好比一座“摩天大楼”，里面有很多的“小房子”，而这些“小房子”里的“住户”就是数据。我们可以利用程序向“大楼”里申请一个“小房子”，并为它贴上对应的“门牌号”，方便我们以后在需要拜访时，更容易找到它。当然，“小房子”里住谁也是由我们决定的，我们可以让它是空的，也可以让它有“常驻住户”(数据保持不变)，或者是“流动住户”（数据可能发生变化）。

```

```

对应到编程中，我们将“流动住户”的“小房子”称为变量，申请“小房子”的过程就是创建变量的过程，而“门牌号”也就是变量的名称，变量的特点就是存放其中的数据是可变的`图示`。与之相对的还有“常量”的概念——“常驻住户”的“小房子”，其中的数据是在程序运行过程中维持不变的。

```

```

我们能够在创趣编程软件中使用变量，与变量相关的操作都在变量模块库中`图示`。软件默认会创建一个变量，名称为“我的变量”。点击“建立一个变量”创建新的变量，在弹出的对话框中，可以完成变量的命名以及变量能够运用的范围（默认“适用于所有角色”即可）。要想删除一个变量或者修改变量名的话，右键点击模块库中变量，在弹出的菜单中选择对应选项即可。`图示`

```

```

变量还可以在舞台区域进行显示，将变量模块库中变量名前方框勾选上时，显示；反之，隐藏。右键点击舞台区的变量名，还可以进行显示形式的切换。

```

```

当我们成功创建了一个变量时，也就拥有了一个“房子”，"房子"里初始“住户”是数字“0”。如果想要向其中安排新的“住户”，需使用` 图示`，编程中，我们也将这一过程称为“赋值操作”（将值赋予变量）。
