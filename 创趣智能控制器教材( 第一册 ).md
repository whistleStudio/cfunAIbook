## 第1节 数据的家 - 变量

无论是在屏幕上显示一段话，还是完成一个复杂的计算，在程序中用到的“话”或者“数字”都是数据的一种表现形式。实际上， 编写程序的过程，就是寻找数据间关系，并用计算机语言表示出的过程。对于程序而言，有些数据可能很庞大且需要反复使用，有些数据可能不确定要依赖于别的数据，并不是所有的数据都会像“你好世界”这样可以很容易地直接表达。那么，有什么更好的办法记录和运用它们呢？

`图示`原来，计算机的大脑就好比一座“摩天大楼”，里面有很多的“小房子”，而这些“小房子”里的“住户”就是数据。我们可以利用程序向“大楼”里申请一个“小房子”，并为它贴上对应的“门牌号”，方便我们以后在需要拜访时，更容易找到它。当然，“小房子”里住谁也是由我们决定的，我们可以让它是空的，也可以让它有“常驻住户”(数据保持不变)，或者是“流动住户”（数据可能发生变化）。

```

```

对应到编程中，我们将“流动住户”的“小房子”称为变量，申请“小房子”的过程就是创建变量的过程，而“门牌号”也就是变量的名称，变量的特点就是存放其中的数据是可变的`图示`。与之相对的还有“常量”的概念——“常驻住户”的“小房子”，其中的数据是在程序运行过程中维持不变的。

```

```

## 第2节 和计算机互动 - 输入与输出

“输入”和“输出”是我们在学习计算机编程时，经常被提及到的两个概念，它其实指的是信息在不同方向的传递过程。谈到方向，我们需要明确的就是它的参照点。方向和参照点最常见的一个例子，就是一列运动的火车，车厢里人看到的窗外景色是向后移动的；而车厢外的人看见的火车则是向前移动的。造成方向不同的原因就是参照点选取的不同，车厢内的人参照的是火车，而车厢外的人参照的则是地面。`图示`

```

```

接下来的讨论，都会以计算机本身为参照点。向计算机进行“输入”操作，也就是从外接环境中读取信息到计算机里，展开后续的运算。`图示`信息的种类允许很多种，可以是一段文字，也可以是一段被计算机麦克风收录的声音，还可以是一张由计算机摄像头拍摄到的图像等。

```

```

“输出”操作指的是计算机向外界环境传递信息的过程，同样信息的形式也是多种多样的，常见的有屏幕的动画，播放的音乐等。

```

```

# 第3章 程序结构

程序结构对整体程序至关重要，从表象上来看，程序由一个个模块构成，而结构则决定了它们的摆放顺序；从本质来看，一段程序的编写是为了实现某种功能，而整体实现的过程是可以拆分成每一小步来进行的，结构则决定这每一小步之间的联系。

```

```

小创和小云最近也在学校学习了高斯算法。他们觉得很神奇，因为对于人而言，运用常规的方法，从1开始一个一个数加到100的确是件非常麻烦的事情，但现在有了高斯算法，就能很轻松地算出累加的和。将信将疑的他们想用逐个累加的结果与高斯算法的结果做对比，验证高斯算法的正确性。
